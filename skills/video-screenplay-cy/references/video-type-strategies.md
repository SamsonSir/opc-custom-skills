# 视频类型识别与处理策略

## 类型识别方法

根据文本内容特征自动判断视频/内容类型，然后应用对应的场景切分策略和分镜风格。

## 五种主要类型

---

### 1. 教程/知识类

**识别信号**：
- 步骤性词汇："首先""第一步""接下来""然后""最后""总结"
- 操作指令："打开""点击""输入""安装""配置""运行"
- 知识讲解："什么是""为什么""原理是""区别在于"
- 编号列表：明确的 1、2、3 结构
- 单人独白为主

**场景切分策略**：
- 按**知识点/步骤**划分场景
- 每个独立的操作步骤 = 一个场景
- 开场介绍 + N 个步骤 + 总结回顾 = 典型结构
- 时间间隔参考：章节间 3-5 秒间隔

**景别倾向**：
- 主讲人讲述 → 中景/中近景
- 软件演示 → 特写（屏幕录制）
- 开场/结尾 → 中景
- 强调要点 → 近景（推）

**运镜倾向**：
- 绝大部分 → 固定
- 强调重点 → 缓推
- 章节结束 → 缓拉

**典型分镜节奏**：1-2 镜/分钟，每镜 20-60 秒

---

### 2. Vlog/旅行类

**识别信号**：
- 地点词汇：城市名、景点名、餐厅名、"来到""到了""现在在"
- 体验描述："好吃""好看""推荐""打卡""体验"
- 第一人称叙述为主
- 频繁的场景/地点变换
- 口语化、随性的表达

**场景切分策略**：
- 按**地点变化**划分场景
- 每到一个新地点 = 一个新场景
- 同一地点内按活动划分子场景
- 时间线一般为顺序（从出发到结束）

**景别倾向**：
- 到达新地点 → 远景/全景（建立场景）
- 探索过程 → 中景/跟镜头
- 美食/细节 → 特写
- 自拍讲述 → 中近景

**运镜倾向**：
- 展示环境 → 摇
- 行走/移动 → 跟
- 美食/物品 → 推（特写）
- 大场景 → 升/降

**典型分镜节奏**：3-5 镜/分钟，每镜 8-20 秒

---

### 3. 访谈/对话类

**识别信号**：
- 问答模式：明确的提问和回答
- 多人发言：2 人及以上的对话
- 称呼使用："老师""你觉得""请问""我认为"
- 观点表达："我觉得""我的看法是""根据我的经验"
- 话题切换通过提问引导

**场景切分策略**：
- 按**话题/问题**划分场景
- 每个核心问题 = 一个场景
- 注意区分主题切换和追问（追问不算新场景）
- 如有多个嘉宾，按嘉宾×话题组合

**景别倾向**：
- 双人对话 → 中景（双人镜头）
- 说话者 → 中近景（单人）
- 倾听者反应 → 近景
- 开场介绍 → 全景/中景

**运镜倾向**：
- 大部分 → 固定（正反打）
- 嘉宾切换 → 摇
- 情感高潮 → 缓推

**典型分镜节奏**：2-3 镜/分钟，正反打交替

---

### 4. 叙事/剧情类

**识别信号**：
- 故事结构：有明确的起承转合
- 动作描述：人物行为、场景变化
- 情感词汇："突然""没想到""终于""可是"
- 时间推进："后来""那天""几年后"
- 可能有多角色对话
- 戏剧冲突和转折

**场景切分策略**：
- 按**戏剧弧/事件**划分场景
- 三幕结构：开端(25%) → 发展(50%) → 结局(25%)
- 每个关键事件/转折 = 场景边界
- 注意高潮和情感节点

**景别倾向**：
- 全范围使用，根据情感需要
- 建立场景 → 远景/全景
- 正常叙事 → 中景
- 情感高潮 → 近景/特写
- 转折时刻 → 特写

**运镜倾向**：
- 紧张/深入 → 推
- 释放/展开 → 拉
- 追逐/跟随 → 跟/移
- 突然转折 → 甩
- 情感表达 → 各种配合

**典型分镜节奏**：3-6 镜/分钟，根据情绪节奏变化

---

### 5. 数据/展示类

**识别信号**：
- 数字和数据："增长了 XX%""达到 XX 万""同比""环比"
- 趋势分析："上升""下降""持平""突破""创新高"
- 对比："相比""而""但是""另一方面"
- 列举："第一""其次""还有""另外"
- 产品/技术介绍

**场景切分策略**：
- 按**主题段/数据维度**划分场景
- 每组相关数据 = 一个场景
- 对比分析前后各自独立
- 总结/结论作为单独场景

**景别倾向**：
- 数据图表展示 → 特写（全屏图表）
- 讲解者解读 → 中景/中近景
- 产品展示 → 中景 → 特写（细节）
- 总结 → 中景

**运镜倾向**：
- 图表展示 → 固定
- 强调某个数据点 → 推
- 从细节到全貌 → 拉
- 产品展示 → 摇/环绕

**典型分镜节奏**：1-3 镜/分钟，数据图表需要停留

---

## 混合类型处理

很多内容是混合型的（如"知识+演示""Vlog+访谈"）。处理策略：

1. **识别主类型**：占比最大的类型作为主类型
2. **识别子类型**：局部段落使用的其他类型
3. **分段应用**：不同段落按对应类型策略处理
4. **保持一致性**：整体风格跟随主类型，局部可变

## 从字幕到场景的通用规则

### 场景边界识别

以下信号表示应该切分新场景：

1. **话题显著变化**：关键词和主题发生转换
2. **时间间隔 > 5秒**：字幕之间出现较长停顿
3. **转折连接词**：但是、然而、不过、接下来、另外、最后
4. **明确的结构标记**：第一/第二、首先/其次、Part 1/Part 2
5. **说话人切换**：从 A 的独白变为 B 的发言（访谈类）
6. **地点变化信号**：提到新的地点或环境（Vlog类）

### 角色识别

从字幕文本中识别角色的方法：

1. **称呼提取**：文本中出现的名字、称谓
2. **语气判断**：不同说话者的用词风格差异
3. **对话模式**：问答结构暗示至少两个角色
4. **自我介绍**："我是XX""大家好我叫XX"
5. **第三方提及**："XX说过""根据XX的观点"（此人为被提及角色）

### 时长估算

当字幕无时间戳时，按以下规则估算：

- 中文口语速度：约 200-250 字/分钟
- 每 100 字 ≈ 25-30 秒
- 含停顿/思考的内容（访谈类）：每 100 字 ≈ 30-40 秒
- 含演示操作：需额外增加 50-100% 时间
