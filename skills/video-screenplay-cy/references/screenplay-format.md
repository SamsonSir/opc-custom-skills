# 中文剧本格式规范

## 基本结构

一个完整的中文剧本包含以下部分：

### 1. 标题页

```
《剧本标题》

类型：[短片/长片/纪录片/教程/...]
时长：约 XX 分钟
编剧：[名字]
日期：[YYYY-MM-DD]

一句话梗概：[Log Line]
```

### 2. 角色表

```
## 角色表

| 角色 | 身份描述 | 年龄 | 出场方式 |
|------|---------|------|---------|
| 张明 | 主讲人，AI工程师 | 30s | 出镜 |
| 旁白 | 解说者 | - | 画外音(V.O.) |
| 李华 | 受访者，产品经理 | 28 | 出镜 |
```

### 3. 场景格式

中文剧本的场景标题格式（注意与好莱坞格式不同，中文是 时间-景别-地点）：

```
### 场景 1

**日/内景 - 办公室**
**时间：00:00 - 01:30**

（镜头缓缓推近一间现代化的办公室，桌上放着一台打开的笔记本电脑）

张明坐在工位前，面对镜头微笑。

张明：大家好，欢迎来到今天的分享。今天我要给大家介绍一个非常有趣的项目。
（停顿，拿起桌上的咖啡杯）

张明：在开始之前，先简单做个自我介绍——
```

### 4. 元素写法

#### 场景标题
```
**[时间段] / [内景|外景] - [地点]**
```
- 时间段：日/夜/晨/黄昏
- 内/外景：内景、外景、内外景
- 地点：具体场所名称

#### 动作描述（Action）
用括号包裹，描述画面中的视觉内容：
```
（张明站起身，走向白板，拿起马克笔开始画图）
```

#### 台词（Dialogue）
格式：`角色名：台词内容`
```
张明：这个方法的核心思路其实很简单。
李华：能再具体解释一下吗？
```

#### 旁白（Voice Over）
```
旁白(V.O.)：在人工智能飞速发展的今天，每个人都在思考同一个问题——
```

#### 画外音（Off Screen）
```
张明(O.S.)：你们看到屏幕上的这张图了吗？
```

#### 括号注释（Parenthetical）
在台词前标注语气或动作：
```
张明：（兴奋地）你看这个结果！准确率达到了 97%！
李华：（沉思）这确实出乎意料……
```

#### 转场
```
【切换】
【淡入】
【淡出】
【叠化】
```

### 5. 特殊标记

- `[推测]`：基于文本推测的画面描述，非原文直接信息
- `[字幕]`：屏幕上显示的文字内容
- `[图表]`：展示图表/数据可视化的画面
- `[演示]`：软件操作/产品演示画面

## 完整示例

```markdown
《AI 编程助手入门指南》

类型：教程
时长：约 8 分钟
日期：2026-03-15

一句话梗概：一位 AI 工程师用 10 分钟带你从零上手 AI 编程助手，掌握提升效率的核心技巧。

## 角色表

| 角色 | 身份描述 | 出场方式 |
|------|---------|---------|
| 张明 | 主讲人，高级AI工程师 | 出镜+画外音 |

---

### 场景 1：开场介绍

**日/内景 - 工作室**
**时间：00:00 - 00:45**

（画面淡入，一间简洁的工作室，背景是书架和绿植。张明坐在办公桌前，面对镜头。）

张明：（微笑）大家好，我是张明。今天这期视频，我们来聊聊 AI 编程助手。

张明：很多同学私信问我，AI 编程工具到底怎么用？值不值得学？
（张明稍微前倾，语气变得认真）

张明：我的答案是——非常值得。它能把你的开发效率提升至少 3 倍。

【切换】

### 场景 2：工具介绍

**日/内景 - 工作室（屏幕录制）**
**时间：00:45 - 03:20**

（画面切换到屏幕录制，显示代码编辑器界面）[演示]

张明(V.O.)：我们先来看看最基础的功能。打开你的编辑器——
（屏幕上光标移动到编辑器）

张明(V.O.)：首先，安装插件。在扩展商店里搜索"AI Assistant"……
（屏幕显示搜索和安装过程）[推测]

张明(V.O.)：安装完成后，你会看到侧边栏多了一个图标。点击它，就可以开始对话了。

【叠化】

### 场景 3：实战演示

**日/内景 - 工作室（屏幕录制+画中画）**
**时间：03:20 - 06:50**

（屏幕录制为主画面，右下角画中画显示张明的表情）

张明(V.O.)：现在我们来做一个实战。假设我要写一个数据处理的函数——
（屏幕上开始输入代码）

张明(V.O.)：你看，我只写了一行注释，它就自动给我补全了整个函数。
（张明在画中画中露出惊喜的表情）[推测]

张明：（兴奋地）对，就是这么简单！

【切换】

### 场景 4：总结收尾

**日/内景 - 工作室**
**时间：06:50 - 08:00**

（画面回到张明正面镜头）

张明：好的，今天的分享就到这里。我们来快速回顾一下——
（张明竖起手指数）

张明：第一，安装和配置；第二，基础对话；第三，代码补全；第四，调试协助。

张明：（认真地）掌握这四个核心功能，你就已经能在日常开发中用起来了。

张明：如果觉得有用的话，别忘了点赞关注。我们下期再见！
（张明挥手，画面淡出）

【淡出】
```

## 三幕结构参考

将内容按三幕划分时的比例参考：

| 幕 | 比例 | 功能 | 对应内容 |
|----|------|------|---------|
| 第一幕（开端） | ~25% | 建立背景、引入主题、吸引注意 | 开场白、问题提出、背景介绍 |
| 第二幕（发展） | ~50% | 展开论述、深入探讨、冲突/转折 | 主体内容、核心论点、实战演示 |
| 第三幕（结局） | ~25% | 总结回顾、呼吁行动、展望 | 总结、CTA、预告下期 |
