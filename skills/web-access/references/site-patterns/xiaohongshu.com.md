## 平台特征

- **架构**: 前端渲染（React），内容通过 API 动态加载
- **反爬强度**: 高，静态抓取几乎不可用
- **登录要求**: 部分内容需要登录，搜索功能必须登录
- **内容加载**: 瀑布流，图片懒加载严重

## 有效模式

### 从首页搜索进入
1. 访问 https://www.xiaohongshu.com/
2. 等待页面加载完成
3. 使用搜索框输入关键词（需登录）

### 直接访问笔记（如已知链接）
- 笔记 URL 格式: https://www.xiaohongshu.com/explore/[NOTE_ID]
- 可通过 browser 工具直接打开

### 图片提取
- 图片 URL 在 DOM 的 `src` 属性中
- 提取前必须滚动触发懒加载
- 使用 `/scroll?direction=bottom` 滚动到底部

## 已知陷阱

| 问题 | 说明 |
|------|------|
| 搜索需要登录 | 未登录时搜索框无法使用 |
| 图片懒加载 | 未滚动的图片显示占位符，不是真实 URL |
| API 限制 | 频繁访问会触发滑块验证码 |
| 内容截断 | 长笔记需要点击"展开"才能看到全文 |

## 推荐策略

对于小红书内容获取，建议直接使用 `browser` 工具而非 `web_fetch`。
