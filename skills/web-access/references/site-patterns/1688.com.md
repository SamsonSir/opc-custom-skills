## 平台特征

- **架构**: 前端渲染（React/Vue混合），大量动态内容
- **反爬强度**: **极高**，有专门的风控团队
- **登录要求**: 必须登录才能查看价格、联系方式
- **风控机制**: 滑块验证码、设备指纹检测、行为分析

## 有效模式

### 1. 正常浏览器登录（推荐）
- 使用标准 Chrome（非 headless）
- 正常鼠标点击、输入（避免脚本化操作）
- 首次登录后保持 cookies

### 2. 登录方式选择
优先级:
1. **淘宝/支付宝扫码**（最安全，风控最低）
2. **1688账号密码**（可能需要手机验证）
3. **短信验证码**（频繁操作会触发限制）

### 3. 降低风控的方法
- 登录后**停留3-5分钟**浏览页面
- 不要频繁刷新
- 避免短时间内大量点击
- 使用真实 User-Agent

## 已知陷阱

| 问题 | 说明 |
|------|------|
| 滑块验证码 | 登录或频繁操作后触发，需人工处理 |
| 价格隐藏 | 未登录时显示"登录查看价格" |
| 联系方式隐藏 | 需登录后才能看到卖家电话/微信 |
| 搜索限制 | 未登录时搜索结果不完整 |
| 设备封禁 | 异常操作后可能封禁设备指纹 |

## 推荐策略

1. **使用 browser 工具 + 已登录 Profile**
   ```bash
   browser action:open url:https://www.1688.com
   # 人工扫码登录一次
   # 后续自动保持登录
   ```

2. **避免触发风控的操作**
   - 不要连续快速点击
   - 不要频繁刷新同一页面
   - 滚动页面时模拟真人（先 eval 看结构，再滚动）

3. **数据提取方式**
   - 优先使用 `/eval` 从 DOM 提取
   - 避免截图 OCR（信息可能不全）
   - 如需多页，间隔 5-10 秒翻页

## 登录状态检查

```javascript
// 检查是否登录
document.querySelector('.member-name') !== null
// 或
document.querySelector('[data-spm="userinfo"]') !== null
```

## 与现有 Profile 集成

1688 可以使用 **淘宝/支付宝账号** 登录，如果已有淘宝登录态，1688 会自动关联。

建议：
1. 先登录淘宝
2. 再打开 1688，自动识别登录态
