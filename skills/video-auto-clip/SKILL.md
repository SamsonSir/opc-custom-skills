# 视频智能剪辑（Video Auto-Clip）

## 概述

通过 AI 视觉分析 + ffmpeg 精准剪辑，自动识别并移除视频中用户指定类型的内容片段，可选对固定角标区域做 **去水印（delogo/遮罩）**，确保输出流畅、衔接自然。

**重要：此技能是通用方法论，不绑定任何特定视频。** 目标内容类型与是否去水印由用户在每次任务中指定，切点通过当次逐帧分析动态确定。

## 前置条件

- 系统已安装 ffmpeg 和 ffprobe
- 视频文件可本地访问

## 能力与模型分层（必读）

不同多模态模型在 **小字 OCR、竖屏海报、叠字、口播与字幕不同步** 上差异很大。执行本技能前应先自检：

| 层级 | 适合做法 |
|------|----------|
| **强视觉** | 可按下文完整流程：1fps 粗扫 → 10fps 精边界 → 直接出 filter 脚本 |
| **弱视觉 / 易漏检** | **必须**启用下文「弱视觉模型补充协议」：ROI 放大、二次扫、证据链、人工确认点，**禁止**在证据不足时宣称「已精确定位」 |

> **原则**：剪辑决策必须能对应到 **具体帧文件 + 一句可核对描述**。若无法做到，应明确标注 **不确定区间** 并请用户给时间点或换更强模型/本地 OCR，而不是用「需要 Whisper」一笔带过并强行交付。

## 弱视觉模型补充协议

在模型容易读错字、漏看角落文字、或混淆「语音 vs 画面」时使用。

### 1. 先分类，再动刀（强制）

对每个疑似命中点，在笔记中标注 **来源类型**（可多选）：

- **A 画面大字/海报/报告图**（占屏、易 OCR）
- **B 软字幕/底部字幕条**（需逐字读）
- **C 仅口播**（画面中无对应大字，剪除依赖时间对齐，弱模型易错）
- **D 角标/水印/UI**（优先 **delogo** 或截片尾，而非整段 CUT）

**禁止**在未区分 A/B/C 的情况下，把「粗扫某秒有敏感词」直接写成 CUT；若判定为 C，应写明 **采用保守区间 + 用户确认** 或 **ASR/人工时间戳**。

### 2. ROI 放大抽帧（提高弱模型可读性）

对「价格条、报告图、底部字幕」等区域，在粗扫命中秒附近 **额外**用 `crop` 放大再抽帧（示例）：

```bash
ffmpeg -ss T0 -t 3 -i "INPUT_VIDEO" -vf "crop=w:h:x:y,fps=10" -q:v 2 "ROI_DIR/r_%03d.jpg" -y
```

`w,h,x,y` 按 720×1280 等分辨率预估（底部字幕多在画面下 1/4）。**同一敏感点至少 2 种尺度**：全帧 + ROI。

### 3. 证据链格式（防幻觉）

每个 CUT/KEEP 边界在交付物中至少给出：

- **证据帧文件名**（如 `zone_5_012.jpg`）
- **一句话客观描述**（「左下角黄底黑字含 XX」），避免笼统「疑似价格语音」
- **不确定度**：`高 / 中 / 低`；若为高，**不得**单独作为唯一依据合并长 CUT

### 4. 口播/无画面文字时的策略

当环境 **无法**运行 Whisper/FunASR 等 ASR 时：

- **允许**：按用户给定的大致时间段做 **保守 CUT**，并列出 **假设与风险**
- **不允许**：虚构「已语音级定位」或把 **未在画面上出现的文案** 当成已验证时间线
- **优先**：请用户提供 **一句「从 X 秒到 Y 秒剪掉」**，模型只负责 **换算为 trim 列表与 ffmpeg**，并做衔接与验证

### 5. 交付前自检（弱模型加强）

在宣称「最终版」前必须完成：

1. **ffprobe** 输出视频时长、流是否正常  
2. **filter_complex** 中 `trim` 区间 **无重叠、无倒序**，且与 KEEP 列表一致  
3. 从输出文件对 **每个拼接点** 抽 3～5 帧，确认无目标内容残留  
4. Windows 通用播放：编码建议 **`-profile:v high -level 4.0`**（见下文物「重编码设置」），避免部分播放器「打不开」

### 6. 失败时如何说

标准话术示例：「当前模型在 **第 34s 报告图** 小字识别上置信度低；已给出 **两种方案**（宽 CUT / 窄 CUT）并标风险；请确认是否采用 **用户指定起止时间** 或换更强视觉模型再审一帧。」

## 工作流程

### 第一阶段：视频探查

1. 使用 ffprobe 获取视频元数据（时长、编码、帧率、分辨率）
2. 确认视频格式可被 ffmpeg 处理

```bash
ffprobe -v quiet -print_format json -show_format -show_streams "INPUT_VIDEO"
```

### 第二阶段：粗扫帧提取与分析

1. 按 1fps 提取关键帧到临时目录：

```bash
ffmpeg -i "INPUT_VIDEO" -vf fps=1 -q:v 2 "TEMP_DIR/frame_%03d.jpg" -y
```

2. **逐批读取帧图像**（每批 10 帧），分析每帧内容：
   - 观察画面中的文字、图表、图片等视觉元素
   - 观察字幕内容
   - 根据用户指定的目标类型判断每帧是否命中

3. **构建时间线标注表**，记录每个需要剪除的时间段：

```
标注格式：
- CUT: 起始秒 ~ 结束秒 | 原因描述
- KEEP: 起始秒 ~ 结束秒
```

### 第三阶段：高精度边界定位（关键步骤）

粗扫仅精确到秒级，**必须**对每个切点边界进行高精度分析：

1. **对每个切点前后各 1.5 秒范围**，提取 10fps 帧（每 0.1 秒一帧）：

```bash
ffmpeg -ss START -t DURATION -i "INPUT_VIDEO" -vf fps=10 -q:v 2 "HIRES_DIR/zone_N_%03d.jpg" -y
```

2. **逐帧查看高精度帧**，确定以下精确时间点：
   - 目标内容（海报/报告/价格字幕等）**首次出现**的精确时间（误差 ±0.1s）
   - 目标内容**完全消失**的精确时间
   - 当前语音/字幕**完整结束**的时间

3. **确定安全切点**：
   - 切入点（KEEP段开始）：目标内容完全消失后 0.1-0.2s
   - 切出点（KEEP段结束）：目标内容首次出现前 0.1s

> ⚠️ **不可跳过此步骤。** 仅凭 1fps 粗扫确定的切点会导致残留帧或语音截断。
> 实际案例：海报/图文过渡动画可在 0.1s 内完成显示，1fps 粗扫无法捕捉到过渡帧。

### 第四阶段：构建剪辑方案

根据高精度边界分析结果，计算保留片段列表：

```
KEEP 片段列表：
1. 0.0s - A秒   (安全区，无目标内容)
2. B秒 - C秒    (跳过了目标内容段)
3. D秒 - E秒    (跳过了目标内容段)
...
```

验证要点：
- 每个 KEEP 段不应包含任何残留的目标内容
- 相邻 KEEP 段的语义应能自然衔接
- 若某个 KEEP 段极短（<0.8s），评估是否有保留价值

### 第五阶段：执行剪辑

1. **（可选）去水印 / 角标遮罩**：若用户要求去除平台角标、ID、导流信息（画面固定区域），可在 `split`/`trim` 之前对整段视频做区域处理。常用 **`delogo`**（需根据分辨率实测 `x,y,w,h`）：

```
[0:v]delogo=x=XL:y=YL:w=WL:h=HL:enable='between(t,0,T1)'[dl1];
[dl1]delogo=x=XR:y=YR:w=WR:h=HR[dv];
```

- **左上角 / 开场信息条**：不少竖屏视频里 **仅在片头几秒** 出现（如抖音号+搜索条叠在画面上方/一角），应对该路 `delogo` 加 **`enable='between(t,0,T1)'`**（例如 `T1=5`），避免全程涂抹、减少画质损失。
- **右下角平台角标**：常见为 **全程固定** 的抖音图标+号+「悦悦家」等，第二路 `delogo` **不加 `enable`**，对整段时长生效。

> **注意（重要）**：部分 ffmpeg 版本中，`delogo` 的矩形若贴边 `x=0`/`y=0` 可能报错 *"Logo area is outside of the frame"*。可将矩形 **整体内缩 10–20 像素**（例如 `x=15:y=15`）再试；或改用 `boxblur`+`overlay`/`drawbox` 等遮罩方案。
>
> **区域手感**：同一分辨率下也需目测微调——**右下角**水印多在底边一条，矩形 **高度 `h` 过大或 `y` 过小** 会把修补区拉得「太高」盖住人物；**左上角**若 **宽度 `w` 不足** 会露字，**高度 `h` 过大** 会浪费画面。以抽帧截图对齐角标边缘为准。

2. 编写 ffmpeg filter_complex 脚本文件（避免命令行解析问题）。视频链使用去水印后的 `[dv]`，音频仍从 `[0:a]` 拆分：

```
[dv]split=N[vin0]...[vinN-1];
[0:a]asplit=N[ain0]...[ainN-1];
[vin0]trim=START0:END0,setpts=PTS-STARTPTS[v0];
[ain0]atrim=START0:END0,asetpts=PTS-STARTPTS,afade=t=out:st=DUR0-0.2:d=0.2[a0];
[vin1]trim=START1:END1,setpts=PTS-STARTPTS[v1];
[ain1]atrim=START1:END1,asetpts=PTS-STARTPTS,afade=t=in:d=0.15,afade=t=out:st=DUR1-0.2:d=0.2[a1];
...
[vinN-1]trim=STARTN:ENDN,setpts=PTS-STARTPTS[vN-1];
[ainN-1]atrim=STARTN:ENDN,asetpts=PTS-STARTPTS,afade=t=in:d=0.15[aN-1];
[v0][a0][v1][a1]...[vN-1][aN-1]concat=n=N:v=1:a=1[outv][outa]
```

2. 执行 ffmpeg 命令：

```bash
ffmpeg -i "INPUT_VIDEO" \
  -filter_complex_script "FILTER_SCRIPT.txt" \
  -map "[outv]" -map "[outa]" \
  -c:v libx264 -profile:v high -level 4.0 -crf 18 -preset medium \
  -c:a aac -b:a 128k -ar 44100 \
  -pix_fmt yuv420p -movflags +faststart \
  "OUTPUT_VIDEO" -y
```

> **兼容性**：部分 Windows 自带播放器对 **H.264 Level 过高**（如 6.x）的 MP4 支持差，表现为「无法打开」。显式 **`-profile:v high -level 4.0`** 可显著减少此类问题；若仍异常，建议用 VLC 试播或用 ffprobe 检查封装是否正常。

### 第六阶段：输出验证

**必须**从输出视频中提取关键帧验证切点正确性：

1. 针对每个切点位置（编辑后时间线），提取前后各 1.5 秒、5fps 的帧：

```bash
ffmpeg -ss CUT_POINT_EDITED -t 3.0 -i "OUTPUT_VIDEO" -vf fps=5 -q:v 2 "VERIFY_DIR/check_%03d.jpg" -y
```

2. 逐帧确认：
   - 切点前后无目标内容残留
   - 字幕与语音一致（无截断字幕配截断语音的情况）
   - 画面衔接自然

3. 如发现问题，返回第三阶段重新精确定位

## 流畅衔接策略

### 音频处理

- 每个片段尾部添加 0.2 秒 `afade=t=out` 淡出
- 每个片段头部（除首段外）添加 0.15 秒 `afade=t=in` 淡入
- 避免音频硬切导致的爆音或突兀感

### 视频切点选择原则

1. **语义断点优先**：在句子结束、停顿处切割，**绝不**在句子中间切割
2. **动作完成处**：在一个动作完成而非进行中切割
3. **场景变化处**：利用自然的场景变化隐藏切割痕迹
4. **避免切断字幕**：确保切点不会截断正在显示的字幕
5. **图文过渡提前量**：图片/海报/报告等覆盖式视觉元素通常在 0.1-0.2s 内突然出现，需通过 10fps 帧精确定位其首帧，切点设在首帧之前 0.1s
6. **语音领先字幕**：语音通常比字幕出现早 0.1-0.3s，当需要切除某段字幕内容时，切点必须在语音开始之前（而非字幕出现之前）
7. **字幕语音一致性**：任何切点处，字幕显示的文字和听到的语音必须一致，绝不允许出现"字幕完整但语音截断"或"语音完整但字幕截断"的情况

### 重编码设置

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `-crf` | 18 | 接近视觉无损的高质量 |
| `-preset` | medium | 编码速度与质量均衡 |
| `-profile:v` / `-level` | `high` + `4.0` | 兼容多数播放器（尤其 Windows 自带） |
| `-pix_fmt` | yuv420p | 最大兼容性 |
| `-movflags` | +faststart | Web 渐进式播放 |
| `-b:a` | 128k | 音频质量与大小均衡 |
| `-ar` | 44100 | 音频采样率常见兼容值 |

## 目标内容类型（按用户指定）

以下为常见可识别内容类型，实际目标由用户在每次任务中明确指定：

| 类型 | 识别特征 |
|------|----------|
| 价格信息 | 数字+元/块/¥/$、"到手价"、"仅需"、具体金额或价格范围 |
| 检测报告 | 检测/检验机构文件、CNAS/CMA标志、报告图片、"机构检测"等 |
| 广告片段 | 品牌LOGO大图、促销标语、购买引导 |
| 二维码 | 画面中出现QR码 |
| 联系方式 | 电话号码、微信号、地址信息 |
| 平台尾卡 | 抖音/快手等平台结尾推广页 |

### 营销/合规敏感词（示例，以用户当次清单为准）

用户可能要求剪除字幕或口播中出现的**整句或整段**，常见包括：

| 类别 | 示例关键词/画面 | 说明 |
|------|-----------------|------|
| 抗菌宣称 | 抗菌、10A抗菌、抗菌效果、抗菌底裆 | 常伴随功能海报或特写 |
| 价格 | 价格、块/元、到手、多少钱、20多块… | 含具体价、模糊价、到手价话术 |
| 便宜 | 便宜、实惠、划算（若用户指定要剪） | 按用户当次定义执行 |
| 食品级 | 食品级 | 若出现在字幕/口播则剪除 |
| 检测报告 | 检测报告、机构检测、MA/CNAS 报告图 | 含报告海报、证书全屏、字幕提及 |

**价格 / 便宜**：若用户明确要求「剪掉价格、便宜相关」，则**只要字幕或语音出现对应词**（即使无具体数字），通常也应切除该句/该段，并与前后 KEEP 段做语义衔接检查。

**仅剪「抗菌」等单一词时**：勿把相邻**无该词**的口播/字幕一并删掉（例如「水洗300次」、按摩颗粒示意图等）；应用 10fps 在 **仅出现目标词** 的区间做窄切，避免「为省事」整段合并删除。

### 平台片尾 / 账号导流

- 常见为**全屏深色底**、头像、**抖音号**、搜索条「抖音搜索」「来抖音发现更多创作者」、底部大 Logo。
- 处理：在 1fps/10fps 中定位**首帧出现片尾 UI** 的时间 `T`，将最终成片时长截断为 **`T - 0.1s ~ 0.3s`**（略早于片尾，避免残留）。若用户说明导流占 **最后 N 秒**，也可用 **`原片总时长 − N`** 作为安全截断点（如 N=3）。

## 注意事项

- 当 CUT 段较多且相邻 KEEP 段很短（<0.8s）时，考虑与相邻 CUT 段合并或直接移除该短片段
- Windows 系统建议将 filter_complex 写入脚本文件，使用 `-filter_complex_script` 参数引用
- 单次 split/asplit 上限建议不超过 10 路，超过时分阶段处理
- 处理长视频（>5分钟）时，先按 0.5fps 粗扫，再对目标区域精扫
- 对包含硬编码字幕的视频，需同时分析字幕文字内容
- **每次执行都是独立分析**：切点、保留片段、剪除内容全部通过当次视觉分析动态确定，不依赖任何预设的固定时间戳
